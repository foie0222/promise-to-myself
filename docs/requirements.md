# 要件定義書: 自分との約束

> バージョン: 1.0  
> 最終更新: 2026-04-09  
> ステータス: ドラフト

---

## 1. プロダクト概要

### 1.1 アプリ名
- **日本語:** 自分との約束
- **英語（App Store用）:** Promise to Myself

### 1.2 コンセプト
自分との約束を1つだけ設定し、期限まで守り抜く。  
余計な機能は一切なし。約束と、自分だけ。

### 1.3 提供する体験価値
- 約束を守りきった時の **達成感・自己肯定感・自己効力感**
- 自分の言葉を尊ぶという行為そのもの
- 約束による行動変容がもたらす成果への気づき

### 1.4 ターゲットユーザー
- 自分を変えたいと思っている人
- 習慣を変えたい・断ちたい人
- シンプルなツールを好む人

## 3. 機能要件

### 3.1 画面構成

#### 画面一覧
| # | 画面名 | 用途 |
|---|---|---|
| 1 | 初回起動 | アプリコンセプトの簡潔な説明 + 通知許可 |
| 2 | 約束入力 | 約束の内容と期限日を入力 |
| 3 | 約束表示（メイン） | 現在の約束と残り日数を表示 |
| 4 | 約束達成確認 | 「守れましたか？」はい/いいえ |
| 5a | 達成演出 | 「はい」選択時 — 宇宙の誕生のような解放感 |
| 5b | 受容フィードバック | 「いいえ」選択時 — あるがままを受け止め、次へ |

#### 画面遷移図
```
[初回起動] → [約束入力] → [約束表示（メイン）]
                ↑                    ↓（期限到来 + 通知タップ）
                |              [約束達成確認]
                |                ↙        ↘
                |        [達成演出]   [受容フィードバック]
                |            ↓              ↓
                └──── [約束入力] ←──────┘
```

### 3.2 各画面の詳細

#### 画面1: 初回起動
- アプリの世界観を伝える短いメッセージ
- 通知許可のリクエスト
- 1回だけ表示（初回起動時のみ）

#### 画面2: 約束入力
- **入力項目:**
  - 約束の内容（テキスト、1行〜数行）
  - 期限日（日付ピッカー、今日以降）
- **バリデーション:**
  - 約束内容は空欄不可
  - 期限日は今日より未来であること
- 「約束する」ボタンで確定

#### 画面3: 約束表示（メイン）
- 現在の約束内容を表示
- 期限日を表示
- 残り日数を表示
- **取り消しボタン**（目立たない配置）

#### 画面4: 約束達成確認
- 期限日の通知をタップ → この画面
- またはアプリ起動時に期限到来を検知 → この画面
- 「自分との約束、守れましたか？」
- 「はい」「いいえ」の2択

#### 画面5a: 達成演出（「はい」選択時）
- 宇宙が解き放たれるような、新しい自分が生まれるような演出
- 光のビッグバン → 星雲の広がり → 新しい宇宙の誕生をイメージ
- タップで約束入力画面へ遷移

#### 画面5b: 受容フィードバック（「いいえ」選択時）
- **責めない。悪いこととしない。ただ事実として受け止める。**
- 静かなトーンで「あるがまま」を承認するメッセージ
- 例: 「約束は果たせなかった。それだけのこと。」
- 次に違いをどう作るかを意識づけるメッセージ
- 例: 「次はどんな約束を、自分と交わしますか？」
- タップで約束入力画面へ遷移

### 3.3 約束の取り消しUX

**意図:** 取り消しは可能だが、自分の言葉の重みを感じてもらう

**フロー:**
1. メイン画面の取り消しボタンをタップ
2. 確認画面表示:
   - メッセージ例: 「言葉は命です。あなたが自分に誓った言葉を、本当に手放しますか？」
3. 「それでも取り消す」「約束を続ける」の2択
4. 「それでも取り消す」を選択 → 約束入力画面へ

### 3.4 通知

| 項目 | 内容 |
|---|---|
| タイミング | 期限日の朝9:00（JST） |
| タイトル | 自分との約束 |
| 本文 | 「約束の期限です。守れましたか？」 |
| 種類 | ローカル通知（UNUserNotificationCenter） |
| 期限前リマインド | なし |

## 4. 非機能要件

### 4.1 パフォーマンス
- アプリ起動: 1秒以内
- 画面遷移: 即時（アニメーション除く）

### 4.2 プライバシー
- ネットワーク通信なし
- App Tracking Transparencyフレームワーク不要
- App Store Privacy Label: 「データ収集なし」

### 4.3 アクセシビリティ
- VoiceOver対応（SwiftUIの標準サポートで基本対応）
- Dynamic Type対応

### 4.4 多言語対応
- v1.0: 日本語のみ
- 将来: Localizable.strings準備だけしておく

## 5. デザイン方針

### 5.1 テーマ
- **黒ベース / 宇宙空間**
- 洗練されたミニマルデザイン
- 静謐で内省的な雰囲気

### 5.2 カラーパレット（暫定）
| 用途 | 色 |
|---|---|
| 背景 | #000000〜#0A0A0A（漆黒） |
| メインテキスト | #FFFFFF（白） |
| サブテキスト | #888888（グレー） |
| アクセント | #C0A0FF〜#8080FF（淡い紫〜青、星のイメージ） |

### 5.3 フォント
- システムフォント（San Francisco）
- 約束テキストは大きめ、中央配置

### 5.4 アニメーション
- 最小限の上品なアニメーション
- 達成演出: 宇宙テーマに沿った光・星の表現
